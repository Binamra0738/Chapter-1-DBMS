Keys in DBMS
(Day 4)

There are many types of keys in DBMS. Some of them are discussed below:


Primary Key

A primary key is a column or a set of columns that uniquely identifies each row in a table. The primary key is chosen by the database designer from among the different candidate keys.


Candidate Key

A candidate key is any column or a set of columns that can uniquely identify each row in a table. A table can have multiple candidate keys.


Alternate Key

An alternate key is a candidate key that is not used as the primary key.


Super Key

A super key is a set of attributes that, taken collectively, can uniquely identify a record in a table.


Foreign Key

A foreign key is a column or a set of columns in a table that references the primary key of another table. It is used to establish and enforce a relationship between two tables.


Composite Key

A composite key is a key that consists of a combination of attributes that can uniquely identify a record.
